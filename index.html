Welcome to Your GitHub Website Tutorial!

As a T Level Digital student, learning to build and host a personal website is a fantastic skill. GitHub Pages allows you to create a free, professional-looking site without any hosting costs. This interactive tutorial will guide you through every step, from signing up to launching your site.

Why GitHub Pages?

  • Free hosting
  • Easy integration with Git for version control
  • Custom domains (optional)
  • Great for portfolios, blogs, or project showcases
Tip: You'll need a computer with a web browser and basic file editing (like Notepad or VS Code, which is free).

Click on a step in the navigation to begin!

Step 1: Create a GitHub Account

To get started, you need a GitHub account. It's free and quick to set up.

  1. Go to github.com.
  2. Click "Sign up" in the top right.
  3. Enter your email, create a password, and choose a username (this will be part of your site URL, e.g., username.github.io).
  4. Verify your email and complete the setup.

Step 2: Create a New Repository

A repository (repo) is like a folder for your website files. For a personal site, name it username.github.io (replace "username" with your GitHub username).

  1. Log in to GitHub and click the green "New" button or go to github.com/new.
  2. Enter the repository name: yourusername.github.io
  3. Make it public (required for free Pages).
  4. Check "Add a README file".
  5. Click "Create repository".

Official guide: GitHub Quickstart

Tip: If you make a mistake, you can always delete and recreate the repo.

Step 3: Add Your Website Files

Now, upload or create your HTML files. Start simple with an index.html file.

  1. In your repository, click "Add file" > "Create new file".
  2. Name it index.html.
  3. Add basic HTML code, e.g.:
<!DOCTYPE html>
<html>
<head><title>My Site</title></head>
<body>
<h1>Welcome to My Personal Website!</h1>
<p>Hello, I'm a T Level Digital student.</p>
</body>
</html>
  1. Commit the file by scrolling down and clicking "Commit new file".
  2. Optionally, add CSS or images by uploading files via "Add file" > "Upload files".

Step 4: Enable GitHub Pages

GitHub Pages will host your site automatically from the repo.

  1. Go to your repository's "Settings" tab.
  2. Scroll to "Pages" in the left sidebar.
  3. Under "Source", select "Deploy from a branch".
  4. Choose "main" branch and "/ (root)", then save.
  5. Your site URL will appear (e.g., https://yourusername.github.io). It may take a few minutes to go live.

Step 5: Customize and Deploy Updates

Make it your own! Edit files directly in GitHub or use Git for advanced control.

  • Add themes: Use free templates from Jekyll Themes.
  • Custom domain: In Pages settings, add your domain (requires DNS setup).
  • Update: Edit files and commit – changes deploy automatically.
